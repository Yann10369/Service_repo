你这里列出的其实是 **四个一级页面**：**首页、相册、AI 分析、我的**。根据文档项目目标，这个 App 的核心是：读取本地相册图片，上传后由 AI 分析照片场景、人物情绪、画面内容，并生成标签、描述、智能分类和自然语言检索能力。

下面按照这四个页面重新分类功能需求，保证底部功能栏不冲突。

# 一、整体页面定位

底部导航建议固定为：

```text
首页 ｜ 相册 ｜ AI分析 ｜ 我的
```

四个页面的职责分别是：

| 页面   | 核心定位      | 主要负责                |
| ---- | --------- | ------------------- |
| 首页   | 智能相册总览    | 展示最近照片、AI 整理概况、快捷入口 |
| 相册   | 照片管理与分类浏览 | 查看照片、分类相册、标签筛选、搜索结果 |
| AI分析 | 图片导入与分析任务 | 批量上传、AI 识别进度、分析结果管理 |
| 我的   | 用户与系统设置   | 权限、隐私、缓存、模型 API 设置  |

这样可以避免“AI发现”“AI分类”“AI分析”混在不同页面中造成冲突。

---

# 二、首页功能需求

首页是用户打开 App 后看到的总览页，重点不是处理具体任务，而是让用户快速看到“智能相册已经帮我整理了什么”。

## 1. 智能相册概览

功能需求：

* 显示已导入照片总数
* 显示已完成 AI 分析照片数量
* 显示待分析照片数量
* 显示今日新增照片数量
* 显示最近一次 AI 分析时间
* 显示智能整理状态，例如“已为你整理出 12 个智能分类”

## 2. 快捷功能入口

首页可以放一些常用入口，但不要承担完整功能。

功能需求：

* 快速导入照片
* 快速进入 AI 分析任务
* 快速进入智能分类相册
* 快速进入自然语言搜索
* 快速进入标签筛选

## 3. 最近照片展示

功能需求：

* 展示最近导入照片
* 展示最近分析完成照片
* 展示最近浏览照片
* 点击照片进入照片详情页
* 显示照片的 AI 标签摘要

## 4. AI 推荐内容

功能需求：

* 推荐“最近生成的智能相册”
* 推荐“高频标签”
* 推荐“可能感兴趣的分类”
* 推荐搜索语句，例如：

  * “海边开心的照片”
  * “聚餐美食照片”
  * “宠物搞怪照片”

## 首页不应该承担的功能

首页不要负责：

* 完整的图片批量选择
* 详细上传进度管理
* 完整标签筛选
* 完整分类管理
* 模型 API 设置

这些应该交给 **相册页、AI 分析页、我的页**。

---

# 三、相册页面功能需求

相册页是用户查看、筛选、分类浏览照片的核心页面。文档中移动端前端负责相册页面开发、图片加载浏览、标签展示、分类文件夹、图片筛选和滑动预览。

## 1. 全部照片浏览

功能需求：

* 展示所有已导入照片
* 支持网格缩略图浏览
* 支持按时间排序
* 支持按上传时间排序
* 支持点击进入照片详情
* 支持左右滑动预览照片
* 支持照片收藏、分享、删除

## 2. 智能分类相册

这是相册页的重要功能，而不是单独叫“AI分类”作为底部栏。

功能需求：

* 按场景分类照片

  * 美食
  * 风景
  * 人像
  * 宠物
  * 聚会

* 按情绪分类照片

  * 开心
  * 平静
  * 搞怪

* 按标签分类照片

  * 旅行
  * 海边
  * 朋友
  * 晚餐
  * 自拍
  * 生日
  * 校园

每个分类需要显示：

* 分类名称
* 分类封面图
* 照片数量
* 分类标签
* 点击进入对应照片集合

## 3. 标签筛选

功能需求：

* 展示 AI 自动生成的全部标签
* 支持选择单个标签筛选
* 支持多个标签组合筛选
* 支持按场景筛选
* 支持按情绪筛选
* 支持按时间筛选
* 显示筛选结果数量
* 支持清空筛选条件

例如：

```text
海边 + 开心 + 朋友
```

系统返回同时满足这些条件的照片。

## 4. 自然语言搜索

文档中明确提出支持自然语言描述检索，例如“去年夏天在海边大笑的照片”“聚餐美食照片”。

功能需求：

* 支持自然语言输入
* 支持搜索照片
* 支持搜索历史
* 支持推荐搜索语句
* 显示搜索结果
* 显示匹配原因

例如用户搜索：

```text
去年夏天在海边大笑的照片
```

系统可以匹配：

```text
场景：海边
情绪：开心
标签：夏天、朋友、大笑
```

## 5. 照片详情页

照片详情属于相册页的二级页面。

功能需求：

* 大图预览
* 左右滑动切换
* 显示 AI 标签
* 显示 AI 简短描述
* 显示场景识别结果
* 显示人物情绪结果
* 显示相似照片
* 支持重新分析当前照片
* 支持用户手动修改标签

## 相册页不应该承担的功能

相册页不要负责：

* 批量上传全过程
* AI 分析任务进度管理
* 模型 API 配置
* 隐私设置

这些交给 **AI 分析页** 和 **我的页**。

# 四、AI 分析页面功能需求

AI 分析页是整个项目的“智能处理中心”。它负责图片导入、上传、AI 识别状态、分析结果管理。

文档中后端职责包括接收图片批量上传、调用 AI 接口、汇总标签和描述数据、数据缓存，并将分类数据返回前端。

## 1. 图片导入入口

功能需求：

* 读取手机本地相册
* 请求相册访问权限
* 支持选择单张图片
* 支持批量选择图片
* 显示已选择图片数量
* 支持取消选择
* 支持重新选择
* 支持开始上传分析

## 2. 批量上传任务

功能需求：

* 显示上传任务列表
* 显示总图片数量
* 显示已上传数量
* 显示上传进度条
* 显示上传成功数量
* 显示上传失败数量
* 支持失败图片重新上传

## 3. AI 分析进度

功能需求：

* 显示 AI 正在分析的图片
* 显示总分析进度
* 显示单张照片分析状态
* 显示当前识别阶段

识别阶段可以包括：

```text
场景识别中
情绪识别中
标签生成中
描述生成中
结果保存中
```

## 4. 分析状态管理

功能需求：

* 等待分析
* 分析中
* 分析成功
* 分析失败
* 重新分析中
* 分析完成

每张图片可以显示对应状态，例如：

```text
第 12 张照片：正在生成标签
第 13 张照片：分析失败，可重试
第 14 张照片：分析完成
```

## 5. AI 分析结果预览

功能需求：

* 显示最近分析完成的照片
* 显示分析出的场景
* 显示分析出的情绪
* 显示自动标签
* 显示图片描述
* 支持点击查看完整分析结果
* 支持重新分析
* 支持确认保存分析结果

## 6. 任务历史

功能需求：

* 查看历史分析任务
* 查看每次任务上传图片数量
* 查看成功数量
* 查看失败数量
* 查看完成时间
* 支持继续分析未完成任务

## AI 分析页不应该承担的功能

AI 分析页不要作为主要照片浏览页。

它不应该重点展示：

* 全部照片网格
* 完整分类相册
* 长时间浏览照片
* 我的账号设置

这些分别属于 **相册页** 和 **我的页**。

# 五、我的页面功能需求

我的页主要负责用户信息、权限、隐私、缓存、系统设置。

## 1. 用户信息

功能需求：

* 显示用户头像
* 显示用户名
* 显示相册使用统计
* 显示已分析照片数量
* 显示智能标签数量
* 显示缓存占用空间

## 2. 权限管理

功能需求：

* 相册访问权限
* 存储权限
* 网络权限状态
* 权限异常提示
* 引导用户开启权限

## 3. 隐私与数据管理

因为项目涉及本地照片和 AI 分析，所以这个模块很重要。

功能需求：

* 是否允许上传图片到云端分析
* 是否仅使用本地分析
* 删除单张照片分析结果
* 清空全部 AI 分析记录
* 清理缓存图片
* 清理标签缓存
* 查看隐私说明

## 4. AI 分析配置

如果项目采用视觉模型 API 调用，可以在“我的”页面提供配置入口。

功能需求：

* 查看当前 AI 分析方案
* 配置视觉模型 API Key
* 测试 API 是否可用
* 设置是否自动分析新照片
* 设置是否允许后台分析
* 设置分析失败是否自动重试

这里不需要做复杂的模型仓库，也不需要模型启停管理。模型侧只需要调用视觉模型 API 对照片进行分析即可。

## 5. 系统设置

功能需求：

* 主题设置
* 缓存清理
* 图片压缩设置
* 通知开关
* 关于应用
* 版本信息
* 反馈入口

# 七、避免功能冲突的命名建议

为了避免生成 UI 时功能混乱，建议四个底部导航名称固定为：

```text
首页
相册
AI分析
我的
```

不要再混用这些名称：

```text
AI发现
AI分类
智能发现
智能整理
```

这些都可以作为页面内部模块名称，而不要作为底部导航入口。

更合理的归属是：

| 功能名称       | 应该放在哪个页面       |
| ---------- | -------------- |
| AI 智能整理    | 首页模块 / 相册模块    |
| AI 分类相册    | 相册页            |
| AI 分析进度    | AI 分析页         |
| AI 识别结果    | AI 分析页 / 照片详情页 |
| 自然语言搜索     | 相册页            |
| 标签筛选       | 相册页            |
| 批量上传       | AI 分析页         |
| 权限与 API 设置 | 我的页            |

这样整个 App 的信息架构会更清晰，也更适合后续生成统一风格的 UI 界面。
